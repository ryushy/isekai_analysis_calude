# キャラクターシート テンプレート（節構成・各表の定義）

既存の正典シート（[mio](../../../../docs/settings/character_sheet_mio.md)／[nagi](../../../../docs/settings/character_sheet_nagi.md)）から抽出した固定フォーマット。
この骨子を埋めて `docs/settings/character_sheet_<name>.md` を作る。**節の順番・見出しは変えない**（節の追加は可）。

---

## ヘッダー（本文冒頭・引用ブロック）

```markdown
# キャラクターシート：氏名（ふりがな）

> ◯◯（主人公/ヒロイン/副キャラ等）の**制作用・即参照シート**（容姿/体型・服装・性格・口調・能力・関係・声/演技の参照）。
> 設計の根拠は 〇〇_core.md（正本＝protagonist_core.md / heroine_core.md など）。本シートはビジュアル・キャスティング・演技・短尺制作のための要約。
> 関連：（対のキャラ・能力・規格書への相対リンク）。
> ステータス：**vX.Y**（YYYY-MM-DD ＋ 反映内容の一言）。
```

- 一文目で**正本との関係**を明示（正本＝設計、本シート＝要約）。
- ステータスは必ず版（vX.Y）と日付を入れる。新規は v0.1。

## 0. 基本（表）

| 列 | 中身 |
| --- | --- |
| 氏名 | フルネーム＋ふりがな。未確定は `[未決]` |
| 年齢/学年 | 数え・所属校など |
| 所属/出自 | 立場・国籍・出自（体制側/野良 等） |
| 立ち位置 | 物語機能（メインヒロイン／相棒AI／ライバル等） |
| 一言 | そのキャラを一行で射抜くキャッチ（主題との関係） |

## 1. 容姿・体型（表）

- **参考画像はシートに含めない**：著者提示の画像はイラストイメージを伝えるための入力。画像を見て読み取った特徴を**文章で記述**し、画像リンク・埋め込みはシート本体に置かない。
- 列：髪／顔立ち／体型／（あざ・目 等キャラ固有）／第一印象。
- canon と画像が食い違う場合は「※色は◯＝canon（画の◯から変更）」のように**どちらが正典か**を明記。

## 2. 服装（表）

- **表層（学校・日常）／署名アイテム／深層（ダンジョン＝コスチューム）**の三層を基本に。
- コスチュームは能力の外在化（[digital_tattoo §6.5](../../../../docs/settings/digital_tattoo.md)）。表層↔深層の落差＝主題の可視化を一言添える。

## 3. 性格・ペルソナ（表＋必要なら小節）

- 声・雰囲気のキャスティング参照があれば引用ブロックで明記（実在人物は雰囲気限定の注記）。
- 列：**表（公）／素（私）／特定相手の前／芯**など面で分ける。
- コメディ機構（ラッキースケベ等）は小節 `### 3.5` に分け、**設計の核・反応の型・節度（健全/年齢相応）・物語機能（緩急の緩）・担当レビュー**を書く。

## 4. 口調・セリフ（表）

- 列：一人称／二人称／基調／崩れ（油断・動揺時）／技名／例セリフ。
- **地の文で口調が判別できるか**を意識（→ editor-tsukikage 観点）。例セリフは3つ程度。

## 5. 能力（表）

- 列：形（光の象徴＝金属でない）／核（何を編集/守る/消すか）／弱点／対比（他キャラとの点vs面 等）／現状（強さの段階）。
- 武器は**力の象徴＝光の具現**として描く（金属武器として描写しない）。

## 6. 関係・今後（表）

- 列：相手／関係。対のキャラ・体制・FAB・ライバル等。
- 信念衝突・将来の火種・温存キャラへの布石を一言で。

## 7. ビジュアルリファレンス仕様（動画制作用・設定画）

> **狙い**：このシートだけで `storyboard`／`seedance` が**一貫したリファレンス画像**を生成できるよう、多角度・小物・色を**言葉で定義**する（著者提示の写真は載せない。読み取った特徴を文章化する）。

- **7.1 多角度（ターンアラウンド）**：同一人物として描けるよう各アングルの要点を箇条書き。
  - 正面／斜め45度／真横（プロフィール）／背面／俯瞰・煽り。各々で「髪の流れ・あざ/小物の見え方・シルエット」を一言。
  - 表情パターン：基調（無表情・凛）／崩れ（照れ・動揺）／戦闘（集中）など3〜5種を言葉で。
- **7.2 小物・署名アイテム（表）**：列＝アイテム／見た目（形・素材感は光の象徴ルール準拠）／意味・機構／登場シーン。署名アイテム（ヘッドホン・弓巻 等）は必ず記載。
- **7.3 カラー指定（表）**：列＝部位／色名／参考カラーコード（#RRGGBB の目安）／備考。
  - 部位＝髪・瞳・肌・表層衣装・コスチューム（深層）・小物。canon の色を最優先（画と食い違えば canon を採用しその旨注記）。
  - 色は**目安**として添える（断定でなく制作の指針）。FAB・武器・能力光は「淡い光／力の象徴」の色味で。
- **7.4 プロンプト用キャラ正典ブロック**：英語1段落で、storyboard の [character-canon](../../storyboard/references/character-canon.md) にそのまま流せる外見記述（髪・瞳・肌・体型・署名アイテム・トーン）。**読める文字/数字/HUD・git 用語・実在人物名は入れない**。

> 7 の各項は §1〜§2（容姿・服装）と矛盾しないこと。矛盾したら §1/§2（正本準拠）を正とし 7 を直す。

## 記述の原則（再掲）

- 正本・台帳・著者提示に**遡れることだけ書く**。無いものは `[未決]`、推測補完は「創作補完」と明示して著者承認を仰ぐ。
- 未開示の核心（[disclosure_ledger](../../../../docs/series/disclosure_ledger.md)）は確定記述を避け「中盤開示」等の注記を付す。
- git 用語は世界観上も非表示。FAB＝淡い光のみ。
