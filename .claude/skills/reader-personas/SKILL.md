# reader-personas — 3読者ペルソナによる原稿レビュー（v2：読み方軸）

> **CVE-0 適応メモ**（naro_novel から移植・本作向け上書き）
> - 原稿＝当面 [docs/manuscript/prototypes/](../../../docs/manuscript/prototypes/)（本格執筆後は `docs/manuscript/book{N}/`）。引数に明示パスを渡せば優先。`all` は [docs/ops/current.md](../../../docs/ops/current.md) の進行中対象。レビュー出力＝`docs/review/`。
> - v1 退避ペルソナ（年齢軸）は移植時に削除済み（v2＝読み方軸の3名のみ）。
> - **本作固有の着眼**：考察勢（縞田）は [docs/series/disclosure_ledger.md](../../../docs/series/disclosure_ledger.md) の開示段階に照らし「まだ知らないはずを語っていないか」も見る。流し読み勢（速水）は冒頭の掴み・バトルの分かりやすさ・難語/造語の有無を、古参（大隈）は主人公とFABの関係・あざ/喪失の情緒の一貫を見る。

なろう系異世界転生連作の原稿に対し、**3人の独立した読者ペルソナ** にそれぞれの視点でフィードバックを返させるスキル。

**v2（チーム編成v2・OPS-17）**：ペルソナの多様性軸を「年齢」から **「読み方」** に再定義した。年齢軸（21/28/47歳）は同じ読み方の濃淡しか拾えず指摘が重複しがち（Tier B ノイズ源）だったため、なろう読者の行動を分ける **読み方の3類型** に各1名を置き、**1人格＝1故障モード** を担当させる。

## 引数解釈

引数 `$ARGUMENTS` を以下の順で解釈する。

| 引数の形式 | 解釈 |
|---------|------|
| `(空)` または `"all"` | 進行中の巻（`docs/ops/current.md` 参照）の本文ファイルすべてを対象 |
| `"prologue"` または `"序章"` | 該当巻の `00-01_*.md` |
| `"ch3"` または `"第3章"` | 該当巻の `03-NN_*.md` 全話 |
| `"ch3-1"` または `"03-01"` | 該当巻の `03-01_*.md` 単話 |
| `"ep"` または `"エピローグ"` | 該当巻の `EP-NN_*.md` |
| `path/to/file.md` のような絶対・相対パス | そのファイル単体（`book1/`〜`book4/` いずれも可） |
| 上記以外の文字列 | `docs/manuscript/` 配下を Glob で検索し、最も近いマッチを使用 |

複数ファイルがマッチした場合は、各ファイルを個別にレビューせず、**全体を通読した上で** 3ペルソナのフィードバックを生成する（ファイルごとに 3 × N のレビューにはしない）。ただしファイルごとに目立った印象があれば、各ペルソナがフィードバック内で個別に言及してよい。

## 実行手順

### Step 1：対象ファイルの確定

1. 引数を解釈して対象ファイル一覧を確定する。
2. Glob または直接の Read で対象ファイルを列挙、存在を確認する。
3. ファイルが見つからない場合は、該当巻ディレクトリを Glob で列挙し、ユーザーに候補を提示してから停止する。

### Step 2：原稿の読み込み

1. 確定した本文ファイルすべてを Read で読み込む。
2. 同じ巻の `writing_style_guide.md`（book1 のものが正本）は **ペルソナとは別の参考情報として** 軽く参照してよい。ただしペルソナ自身がガイドを読んだ前提にはしない（読者はガイドを読まない）。
3. **ペルソナは内部資料（骨子・設定書・台帳）を読んでいない前提を厳守する**。考察勢（紬）の整合性チェックも「本文に書かれたことだけ」の突き合わせで行う。内部資料との照合は設定考証（台帳番）と編集長の仕事（ゲート1／Tier 統合時）。

### Step 3：3ペルソナのプロファイル読み込み

以下の3ファイルをすべて Read で読み込み、各ペルソナの設定・口調・フィードバック構造を完全に内面化する。

- `.claude/skills/reader-personas/personas/ren_skimmer.md`
- `.claude/skills/reader-personas/personas/tsumugi_theorist.md`
- `.claude/skills/reader-personas/personas/shizue_oldfan.md`

### Step 4：3ペルソナのフィードバック生成

各ペルソナに **完全に憑依** して、それぞれのプロファイルに定められたフィードバック構造に従ってレビューを書く。

**重要な原則**：

- **3ペルソナは互いに干渉しない**。各レビューは独立して書く。「他の人はこう思うかも」のような相互言及は禁止
- **各ペルソナの口調・文体を厳密に守る**。蓮は短文の常体（「〜っす」混じり）、紬は証拠引用つきのです・ます調、静江は柔らかいです・ます調
- **各ペルソナは自分の担当故障モードを最優先で報告する**。蓮＝離脱ポイント、紬＝整合性の疑義、静江＝キャラ・関係性の気がかり。**0件なら「なし」と明言する**（書かないのではなく、検査して無かったと言う）
- **過剰な賛辞・過剰な辛辣さを避ける**。読者として誠実なフィードバックを返す
- **該当箇所への具体的言及を必ず含める**。「全体的に良い」では不十分
- **改善提案は建設的に**。ただし読者なので「こうしたら？」は読者の願望として書く（編集者の改修指示はしない）
- **設定原則（神紋・リン伏せ・多元宇宙隠蔽・staging）への言及は、読者として違和感を持ったときだけ**。著者目線のメタ言及はしない（ペルソナは原稿しか読んでいない）

### Step 5：出力フォーマット

以下の順で、すべてのフィードバックを **1つの応答の中で連続して** 出力する。

```markdown
# 📖 読者ペルソナレビュー：[対象ファイル名 or 章名]

> 対象ファイル：[ファイルパス一覧]
> 総文字数：[概算]字
> 3人の読者ペルソナ（読み方軸 v2）による独立したフィードバック

---

[速水蓮（流し読み勢）のフィードバック全文]

---

[縞田紬（考察勢）のフィードバック全文]

---

[大隈静江（古参ファン）のフィードバック全文]

---

## 🪞 編集長への申し送り（任意：3ペルソナの観点を横断する所感、Claude 視点）

[3〜5文。3ペルソナのフィードバックを俯瞰し、共通項・対立項・Tier 統合のヒントを簡潔に]
```

最後の「編集長への申し送り」は **Claude 自身の立場** で書く。ペルソナでも著者でもない第三者として、3ペルソナの感想を俯瞰した気付きを 3〜5 文で添える。これは省略してもよいが、書く場合は **3ペルソナそれぞれが触れていない切り口** を提示すること（重複させない）。

**v2 の Tier 統合における注意**：3ペルソナは観点が直交するよう設計されているため、v1 よりも「3者一致（Tier A）」は出にくくなる。一致しないこと自体は正常。**各ペルソナの担当故障モードでの単独指摘は、軸の専門性ゆえに Tier B でも重く扱う**（編集長判断）。

## 並列化のヒント（任意・大型原稿向け）

対象ファイルが合計 **2万字を超える** 場合、または同時に複数ペルソナで深く読み込みたい場合は、**3つのペルソナを並列の Agent として走らせる** ことを検討する。

並列化する場合の手順：

1. 対象ファイルのパス一覧を確定
2. 3つの Agent を `general-purpose` で並列起動し、それぞれに以下を渡す：
   - そのペルソナのプロファイルファイルパス（Read で読ませる）
   - 対象原稿のファイルパス（Read で読ませる）
   - フィードバック構造に従って生成し、結果を返すよう指示
3. 3つの Agent の出力を受け取り、Step 5 のフォーマットに整形して出力

並列化しない場合は、メイン Claude が3ペルソナを順に演じる（1万字程度の原稿ではこれで十分）。

## このスキルが扱わないこと

- **校閲・誤字脱字・機械検出可能な事故**：レビュー前に `python scripts/check_manuscript.py <対象>` を先に実行する（CC v2 ゲート2 の検査順序：機械検査 → 読者パネル＋文芸編集 → 編集長統合）
- **内部資料との照合**：台帳（`docs/series/story_timeline.md`／`canon_log.md`／`disclosure_ledger.md`）との突合は設定考証（台帳番）の仕事。紬の疑義はその入力になる
- **書き直し・リライト**：このスキルは感想を返すだけで、本文の書き換えはしない
- **編集者目線の構造分析**：`editor-kawahara`（構造・ゲート1）／`editor-tsukikage`（文芸・ゲート2）が担当
- **集客・ブクマ観点の評価**：`serial-strategist`（ゲート3）が担当
- **ペルソナの追加・変更**：ペルソナを増やす・差し替える場合は `personas/` 配下にプロファイルを追加し、SKILL.md のペルソナ一覧と引数解釈を更新する
