# エピソードID命名規則（連作横断）

> 全10巻のエピソードを一意に識別するプロジェクトID規則。**物語全体→大弧→巻→局面→話** の情報アドレス。
> **設計と執筆の境界（2026-06-15 著者方針・§1.5）**：計画は **巻 → 局面（ブロック）** まで。**話は執筆しながら割る**（1局面に1本以上）。
> 関連：[../plot/series_arc_draft.md](../plot/series_arc_draft.md)（全10巻構成）／[canon_log.md](canon_log.md)／[disclosure_ledger.md](disclosure_ledger.md)／[../ops/README.md](../ops/README.md)。

---

## 0. 一目で分かる版

| 項目 | 値 |
|---|---|
| **ID形式** | `B{巻}-{話2桁}` |
| **例** | `B1-03`（第1巻 第3話） |
| **巻番号** | `B1`〜`B10`（全10巻・3大弧） |
| **特殊話番号** | `00`＝序話／`01`〜`NN`＝通常話／`EP`＝エピローグ／`IL`＝幕間 |
| **ファイル名** | `{話2桁}_{slug}.md`（例：`03_frozen_rain.md`） |
| **巻ディレクトリ** | `docs/manuscript/book{N}/`（執筆開始時に作成） |
| **試作（巻未確定）** | `docs/manuscript/prototypes/` に slug 名で置く（例：`prototype_ep1_battle.md`）。本採用時に B1-... へ昇格 |

> **2026-06-15 改定**：旧 `B{巻}-{章2桁}-{話2桁}`（3階層）を **`B{巻}-{話2桁}`（2階層）** に簡略化。**「章」は廃止**（試行錯誤の間は使わず、LN単行本化のときに数話を束ねる単位として復活させる）。旧 `B1-01-01`〜`B1-04-01` は `B1-01`〜`B1-04` にリネーム済み。

## 1. 階層と責務（情報コントロールの単位）

| 階層 | アドレス | 正本ドキュメント | 何を管理するか | いつ決める |
|---|---|---|---|---|
| **物語全体（連作）** | 連作 | [../plot/series_arc_draft.md](../plot/series_arc_draft.md) | 3大弧・敵の階段・続編(upstream)・通しテーマ | 構想 |
| **大弧（部）** | 大弧I/II/III | series_arc §大弧表 | 弧ごとの焦点・登る敵・開示の山（3〜4巻ぶん） | 構想 |
| **巻（Book）** | `B{N}` | `docs/plot/book{N}_skeleton.md` | 巻の目的・局面・到達点・隠蔽上限 | 構想 |
| **局面（ブロック）** | 巻内の局面名 | book{N}_skeleton §章ブロック | 巻内の山場の流れ（各巻3〜5局面）・伏線の出し入れ | 構想 |
| **話** | `B{N}-{話}` | `{話2桁}_{slug}.md` | 本文。1話＝1公開単位（約6,000字） | **執筆しながら** |

- **開示の可否は必ず [disclosure_ledger.md](disclosure_ledger.md) で確認**：その話の「現在の開示段階」を超える描写は NG。
- **確定事実は [canon_log.md](canon_log.md)** に追記（IDで紐づけ）。

## 1.5 設計と執筆の境界（2026-06-15 著者方針）

- **計画する層＝巻 → 局面まで**。「この局面でここまで到達する」を設計で決める。
- **話は執筆しながら割る**。1つの局面を書いてみて、約6,000字ごとに自然に切れる単位を「話」として確定（1局面に1本以上）。
- **plot（設計メモ）と manuscript（本文）は同じ"話"の2ビュー**（親子の階層ではない）：
  - `docs/plot/ep{N}_plot.md` … 話のプロット骨子（先に書く設計メモ）
  - `docs/manuscript/book{N}/{話2桁}_{slug}.md` … 話の本文
  - 当面 `ep{N}` ＝ `B{巻}-{話}` の作業上の通称（ep1 ↔ B1-01）。正式IDは `B{巻}-{話}` 側。

## 2. 表示書式

- 通常：`B1-03`。
- メディア案件との対応：ストーリーボード案件 slug（[../../media/](../../media/)）には対応する話IDを案件 README に明記（例：`isekai-cve0-battle-rain` ↔ B1-03 想定）。
- 連作通し話数（任意）：必要なら `ep.NNN`（3桁ゼロ埋め・全巻通し）を併記してよい。現時点は未公開のため必須としない。
