# 開示台帳（disclosure ledger）

> **隠蔽事項と伏線の「誰が・読者が・いつ知ったか」の単一正本**（naro_novel から移植）。
> CVE-0 の「X-伏せ開示スケジュール」（[../plot/series_arc_draft.md](../plot/series_arc_draft.md)）を巻別・項目別に台帳化したもの。
>
> **運用ルール**：
> 1. 各項目の「現在の開示段階」を**超える描写は、アウトライン・本文どちらでも NG**（レビュー・編集判定の根拠）。
> 2. 執筆前（章アウトライン時）と執筆後（本文確定時）に必ず照合する。考察勢レビュー（[reader-personas](../../.claude/skills/reader-personas/SKILL.md)）も本台帳で「まだ知らないはずを語っていないか」を見る。
> 3. **git 用語・深層機構**は最大の隠蔽対象（[../settings/battle_system.md §5.5 相当](../settings/battle_system.md)）。表層では現象・日常語のみ。
> 4. 開示が進んだら行を**更新してよい**（現在状態を表す台帳。canon_log と違い可変。更新時は改訂履歴に1行残す）。
