# AI 執筆の癖チェックリスト（参考情報）

> AI で本文を書くと出やすい「機械的な事故＝癖」のエッセンス。naro_novel の原稿リンタから抽出し CVE-0 向けに移植。
> 機械検出は [../../scripts/check_manuscript.py](../../scripts/check_manuscript.py)、文体の正本は [writing_style_notes.md](writing_style_notes.md)、開示の正本は [../series/disclosure_ledger.md](../series/disclosure_ledger.md)。
> 使い方：人格レビュー（reader-personas／editor-*）の**前**に機械検査で事故を洗う。

---

## 1. なぜ「AI の癖」が出るのか

AI は設計資料・分析の語彙や Markdown 記法を、そのまま物語本文に持ち込みやすい。
作者視点・構造説明・強調記号・難語が**地の文に漏れる**と、没入が切れチープになる。
**本文は登場人物の体験だけで構成し、作者の分析語彙を持ち込まない**のが原則。

## 2. チェック項目（重大度つき）

### NG（出たらほぼ事故）

| # | 癖 | なぜダメか | 直し方 |
|---|---|---|---|
| 1 | **git 用語の表層露出**（commit/repo/push/merge/deploy/origin 等） | 本作最大の禁忌（著者要件B）。世界の機構は隠す | 現象・日常語で書く（例：deploy→「降らなかったことになった」） |
| 2 | **メタ作者用語**（主人公/ヒロイン/伏線/クライマックス/本作/地の文/一人称） | 作者視点の漏れ＝物語世界の外側 | 人物名・体験で書く |
| 3 | **章番号参照**（「第二章で」） | 本文に構造を書かない | 場面・時間表現に置換 |
| 4 | **地の文の強調記号**（`**…**`） | 小説本文に Markdown 強調は入れない | 語順・描写で強調する |
| 5 | **異文字混入**（ハングル/キリル/タイ等） | 変換・生成事故 | 削除・修正 |

### WARN（文脈により可・要目視）

| # | 癖 | 着眼 |
|---|---|---|
| 6 | **難語・古風語**（残滓/邂逅/慟哭/黎明/矜持 等） | ライトな現代口語か（[writing_style_notes.md](writing_style_notes.md)） |
| 7 | **開示台帳の早出し語**（偽origin/マスターオリジン/upstream/Mythos/心意/Fable 等） | その話で出してよい段階か（[disclosure_ledger](../series/disclosure_ledger.md)） |
| 8 | **三点リーダ奇数連**（「…」） | 「……」偶数連が標準 |
| 9 | **ダーシ運用**（単独行・会話直前・奇数連） | 演出の癖を抑える |
| 10 | **半角感嘆・疑問符**（!? ） | 本文は全角（！？） |
| 11 | **読点過密**（一文に6箇所以上） | 一文の読点は5まで目安 |
| 12 | **長文**（一文80字超） | 短文志向（ライト口語） |
| 13 | **短行3連続**（一語改行の連続） | 詩的演出過多を避ける |
| 14 | **Markdown リンク残骸**（`［text］（url）`） | 本文には残さない |
| 15 | **呼称ゆれ**（同一人物に〇〇先輩／〇〇さん 等が混在） | **話者で変わるのは正常**だが、同一話者→同一対象のブレ・表記ゆれ（様/さま）は統一。正本＝[../series/appellations.md](../series/appellations.md) |
| 16 | **専門語・予備知識が要る語**（庇/ひさし/あふれ/心意/演算/容量 等） | 設定を知らないと読めない語。現象・体感語へ（例：庇→傘いっぽんぶんの薄い膜／あふれる→抱えきれず潰れる） |
| 17 | **英字（ラテン文字）の混入**（例：small） | 英単語の紛れ込み。日本語に（固有名は header/メタで管理） |

## 2.5 本文セルフチェック（機械化しにくい・目視／レビューで拾う）

> 機械検出が難しいが、これまで著者が指摘した事故。本文確定前に**目視**で、または人格レビューで拾う。

| 観点 | 確認すること | 出典（過去指摘） |
|---|---|---|
| **比喩・固有概念の伝達** | 能力やシステムの比喩を、**初出で効果を具体に見せて**意味が通っているか（「自分に向かって書いた」だけにしない） | 「書いた」が伝わらない |
| **心理状態と描写トーンの一致** | パニック・消耗・平静など、人物の状態に描写の語り口が合っているか（パニック中に冷静な分析をさせない） | 逃走中の分析が不自然 |
| **同語の多義・近接衝突** | 同じ語を別の意味で近くに使っていないか（例：「口」＝入り口／＝言葉） | 「口」の混同 |
| **移動・時系列の自然さ** | 場所の行き来・所要・距離に無理がないか（実在地の動線） | 山手→みなとみらいの不自然 |
| **密度（あっさり防止）** | 各シーンに四要素（五感・内面・会話・戦闘の手触り）があるか／字数は目安内か（[writing_style_notes §4.5](writing_style_notes.md)） | 全体的にあっさり |

## 3. 運用

```bash
# 本文ファイル（または prototypes/）を機械検査。NG が1件以上で終了コード1
python scripts/check_manuscript.py docs/manuscript/book1/01-01_xxx.md
python scripts/check_manuscript.py docs/manuscript/prototypes/
```

- **「## 検証メモ」「## 改訂履歴」等の作者ノート節以降は自動で検査対象外**（本文のみ検査）。
- NG＝直す。WARN＝目視で判断（特に 6 難語・7 早出し語は文脈次第）。
- 機械検査でゼロにしてから人格レビュー（[../review/README.md](../review/README.md)）へ進む。
