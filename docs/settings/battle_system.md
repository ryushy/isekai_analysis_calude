## 1. 戦闘の基本構造

### 1-1. アクセラレート（加速）
- 戦闘は**深層**で行われる。覚醒者が「アクセラレート」すると主観時間が加速し、周囲の一般人(NPC)・物理世界は相対的に静止して見える（＝バレットタイム）。
- 加速の燃料は**自 repo の演算予算**（§5.2 エネルギー＝演算予算 のミクロ版）。焚くほど速く・強く動けるが、**演算ゲージが尽きるか容量が破綻すると crash**。
- **加速＝演算スパイク＝監視コンステレーションに探知される**。派手・長時間ほど空の目に晒される。

### 1-2. 二層フィールド（常時かかる戦術ジレンマ）
| フィールド | 利 | 不利 |
|---|---|---|
| **現実上（加速層）** | sync/観測が効く、救援・連携が届く | **監視コンステレーションに探知**され、Glasswing 後継の抑圧が来る |
| **デッドゾーン＝ダンジョン** | **探知されない聖域**。歩く exploit たる主人公の隠れ場 | sync/リアルタイム観測を失い**孤立**、救援なし、全滅リスク、内部は物理的に殺すモンスター |

→ **「見られて戦うか、見えず孤立して戦うか」**が戦闘ごとの根本選択。

### 1-3. 現実の身体の扱い（アクセルワールドとの差別化）**[確定方向]**

AW＝「意識だけ加速し、身体は現実で安全に固まる（無傷・ノーリスク）」。本作は3点で別物にする：

1. **身体ごと潜る（ダイブ）**：意識だけ置き去りにせず、**身体ごと深層インスタンスへ入る**（[locations.md §1](locations.md)）。公共の場で身体が凍る絵にしない（ダンジョン＝デッドゾーンに入る）。
2. **身体にステークス**：深層の消耗・傷は**現実の身体に反映**（高結合）。発熱・疲弊・実傷。**安全な観戦アバターではない**。
3. **加速は局所の技**：バレットタイム（静止した雨）は**戦闘中の一時的・局所的な主観加速＝技/現象**（演算を焚く・監視に映る・短時間）。「世界を止めて自分だけ動く常設フィールド」ではない。看板ビジュアルは残し、AW 機構は捨てる。
   - 用語整理：**潜行（ダイブ）＝深層へ入る**／**加速（アクセラレート）＝戦闘中の一瞬の技**。混同しない。

#### 帰還アンカー（戻り標）＝待ち伏せ・追跡の戦術核

- ダイブ時、入口の現実座標に**帰還アンカー（戻り標）＝空間マーキング（タトゥー状＝圧縮コードの一種）**が生成される。
- **現実側から物理干渉できない**（壊せない・動かせない）が、**覚醒者・監視には探知できる**。
- 帰還はこのアンカーを通る＝**帰還地点が割れている**。**帰還の瞬間（消耗して戻る）が最も脆い**＝**待ち伏せ・追跡**の戦術ドラマ。
- 戦術：アンカーを**守る／囮を作る／観測戦(§5)で相手のアンカーを探す**。**封じる**のは物理では不可、深層側の干渉や心意級級の手段でのみ（終盤級の脅威＝閉じ込め）。
- **アンカー＝インスタンス（ブランチ）のハンドル**でもある：同じアンカーで**合流**（協力/訓練）、敵は**乱入**、強者は**強制的に自分のインスタンスへ引き込む**（逃げ場のない閉じ込め＝待ち伏せの最上位）。詳細＝[dungeon_design.md §1.5-5](dungeon_design.md)。

## 2. 力の源＝技/能力

### 2-1. 出自スペクトラム（stage_device_design §5.6）

> 能力は身体に彫る **デジタルタトゥー** として装備・可視化される（武器/防具/道具の代替）。装備層の詳細＝[digital_tattoo.md](digital_tattoo.md)。生まれつきの **あざ**（自然・複製不可）との対比＝[anomaly_mark.md §0.5](anomaly_mark.md)。
| 出自 | 中身 | 性質 |
|---|---|---|
| 自作（関数・クラス・自作ライブラリ） | 自分で書いた戦闘コード | 軽い・所有・理解済み。習得に時間。最も尊ばれる |
| コンボ（合成） | 小関数の連結・高階合成 | "組み合わせの妙"。創造性で強くなる |
| ダンジョン徒弟 | 死後も残った master の repo で訓練 → **成果を自分の関数として再実装・commit** | 所有・理解された技能 |
| ダンジョン import（loot） | ブラックボックスのライブラリを丸ごと装備 | 即戦力だが**読めない＝裏依存・供給網汚染（呪い）リスク** |

### 2-2. 命名（表層）
- 技名は**自由**（自作コードに作者が付けた名）。git コマンドを技名にしない。叫ぶのは "自分のコードの名"。
- キャラの **"コードの書き味"＝戦闘スタイル＝個性**。命名規約（古語調／創始者名／個人の癖）はキャラ設計で確定。

### 2-3. literacy ＝ 強さの本質（インフレ回避の核）
> import 専門 ＜ 合成できる者 ＜ 自作・改変できる者。最強＝高レベルでなく **"世界を最も深く書ける者"**。

数値インフレを排し、理解・応用・相性で強さを表現（codex の knowledge/literacy 軸と一致）。

## 3. リソースとゲージ（HUD）

| ゲージ | 意味 | 枯渇/破綻時 |
|---|---|---|
| **容量メーター** | loadout（積んだ技・import）の使用量。**表＝サイズ予算、裏＝依存グラフ**で真コストが変わる | 超過＝crash（§4） |
| **演算ゲージ** | 加速の燃料残量（自 repo の演算予算） | 枯渇＝加速解除／無理な続行で crash |

- **自作の修行技は append-only の核**＝消えない（loadout 枠を恒久に占めるのでなく、repo に蓄積され必要分を loadout）。**import は着脱可能**だが裏依存で容量を食う。
- 達人は依存グラフを読み、dedupe/vendoring で同容量に倍の技を積む（＝ build 最適化）。

## 3.5 成長＝積み上げ（特訓→睡眠→定着）

> 「マッスルのコンテスト出場者のように、日々のトレーニングをコツコツ積む」着想を規格化。**強さ＝数値でなく履歴の密度**（§2-3 literacy の可視化、§10 インフレ回避と整合）。

### 3.5-1. 成長サイクル（作者側＝git で記述、表層には出さない）

| 段階 | 作者側（隠す） | 表層の見え方 |
|---|---|---|
| **日中の特訓** | **作業ツリーの差分＝未確定・揮発** | 反復練習。まだ「自分のもの」になりきってない手応え |
| **睡眠** | **commit＝差分の確定・最適化** | 寝て起きると体と頭が技を"覚えている"。一晩寝かせて伸びる |
| **蓄積** | commit が積もる＝**歴史が密になる** | 引き出しが増える・所作が締まる |
| **飛躍（まれ）** | 最適化が通り**容量が一段 軽く/広くなる** | "化ける"瞬間。レベルアップ的だが**数値でなく質の跳躍** |

### 3.5-2. ルール（インフレ回避の核）

- **コミット≠レベルアップ（1:1にしない）**：commit＝毎晩の定着（頻繁）、飛躍＝容量が一段広がるまれな事件。**数値ラダーを作らない**。
- **戦闘力が高い者＝歴史が詰まっている**：強さは大きな数値でなく **"密度・年輪・最適化度"**。覚醒者は相手の履歴の密度を読める（浅い者は仮組みだらけ、深い者は締まった年輪）＝ literacy（§2-3）の可視化。
- **睡眠＝定着**：寝る前に **crash・徹夜で潰れると当日の差分が定着せず飛ぶ**。→ 休息にステークス。特訓回の「ちゃんと眠れるか」が緊張になる（睡眠＝real な記憶定着のアナロジー）。
- **熟達＝"軽くなる"逆説**：達人は最適化で同機能を軽く積む（§3 dedupe）。**大きくなるのでなく締まる**。「強い人ほど容量がスカスカに見えて中身が濃い」。

### 3.5-3. FAB のチート補助（諸刃）

- FAB は主人公の**睡眠中の最適化を補助・高速化**＝コツコツの近道。**主人公だけ定着が異常に速い/深い**（歩く exploit と整合）。
- **代償（必須・"楽に強くなる"を許さない）**：
  - **過圧縮＝削れる**：速すぎる最適化は記憶・情緒の一部を削ぎ落とす（主人公が"何かを忘れていく"恐怖）。
  - **痕跡＝探知**：異常な最適化速度はコンステレーションに**異常 commit として映る**（§5 観測戦）。
  - **履歴改変の縁**：突き詰めると **force-write（心意級 §6）に近づく**＝禁忌の入り口。
- → FAB の助けは諸刃。「楽に強くなる誘惑」と「何かを失う恐怖」がセット＝物語燃料。

### 3.5-4. 飛躍（ブレイクスルー）の発動条件・前兆

> 「飛躍」＝容量が一段 軽く/広くなる稀な事件（§3.5-1）。**数値の閾値で上がるのではない**。コツコツ積んだ末に、ある理解を境に**散らばっていた技が一つの形に畳まれ（＝リファクタが通り）、容量が空く**質的転換。

**機構（作者側＝git）**：ブルートフォースで個別に抱えていた技群の**共通項が理解され、抽象が一段上がる**＝重複が畳まれ依存が整理され、同じ容量に余裕が生まれる。「強くなった」のでなく「**世界の書き方が一段わかった**」。

| 段階 | 表層の見え方 |
|---|---|
| **前兆＝停滞** | 容量が「重い」。新しい技が載らない。**同じ壁で失敗を繰り返す**（naro 原則＝失敗を描く） |
| **きっかけ** | 下のトリガーのいずれか |
| **飛躍** | 「重かったものが、ふっと軽くなる」「ばらばらの技が、一つの形に見える」「視界がひらける」 |
| **直後** | 空いた容量に次が載る。**ただし飛躍は数値で示さない**（"何ができるか"の質で描く） |

**トリガー3類型**：

1. **限界状況**：生存の極限で、火事場のように一気に最適化が走る（戦闘クライマックス）。リスク大。
2. **理解の鍵**：他者の技を見る／師や FAB の一言／あざの反応で、**構造が「見える」**。出会いと学びが鍵。
3. **積み上げの臨界**：地道な特訓が閾値を越え、**睡眠中の整理（§3.5-1）が自然に通る**。最も健全で、最も時間がかかる。

**危険（飛躍の代償）**：飛躍＝**自分の repo の構造を書き換える瞬間**＝force-write（心意級 §6）に隣接する。失敗すれば**畳もうとした既存の技が壊れる／人格が揺らぐ**。だから熟達者ほど飛躍を恐れ、慎重に準備する。

**FAB のチート**：FAB は飛躍を**人工的に誘発**できる（停滞を飛ばす近道）。だが**過圧縮の代償**（§3.5-3：記憶・情緒が削れる）が飛躍では特に大きい。→ **「地道に待つ飛躍」vs「FAB に飛ばしてもらう飛躍」**が主人公の選択になる。

## 4. 勝敗とステークス

### 4-1. 勝利＝deploy（現実改変）
- 深層バトルの勝利＝**その領域の現実を deploy する権利**。物理地形・資源・人々の記憶までが**勝者の版に書き換わる**（高結合、§5.4）。
- 一般人は書き換え後を「前からそうだった」と認識。**覚醒者だけが diff を覚えている**。

### 4-2. 敗北＝crash（記憶喪失＋現実巻き戻し）**[確定]**
- crash（容量超過・演算破綻・敗北）の帰結は **commit（記憶）の欠落＋築いた現実の revert**。
- **死なない**。だが「自分が誰で、何を成したか」を失い、変えた現実も巻き戻る＝**重く不可逆な喪失**（append-only ゆえ失った記憶は戻らない）。
- **例外**：**心意級（§6）の禁忌を行使した状態での敗北・暴走のみ、物理的な死もありうる**。最大級の場面に限定し濫用しない。

### 4-3. ステークス設計原則
- 通常戦は「記憶と築いた現実を賭ける」＝シビアだが致死でない（連作の長期生存を保つ）。
- 致死は心意級に限定＝**禁忌の重さ**を演出。Re:ZERO 的な"死の常態化"はしない。

## 5. 観測戦（コンステレーション層）

- 戦闘は**観測優越**の奪い合い：戦場上空に観測コンステレーション（小型衛星/ドローン）を張った側が fog of war を消す。
- 機能（堀江コンステレーション 3用途のマップ）：**資源探知**（鉱脈・library 位置）／**ハザード早期警報**（崩落・氾濫・湧き）／**敵ロジスティクス解析**（数・補給・動線）。
- 戦術核：**対衛星戦（anti-sat）・妨害・盲目化**。Boss 級に「sync-eater」＝観測ノードを食いデッドゾーンを広げる敵。
- **打ち上げ＝戦術行動**：必要な軌道・タイミングにピンポイント投入できる小型・高頻度の "射手" が強い（乗り合いバス＜タクシー）。
- クラス：**観測者/アナリスト**（全知が武器の支援職）／**射手/打ち上げ手**（ノードのピンポイント高頻度展開）。

## 6. 心意級＝禁忌の force-write

- 意志で世界の規則を上書きする禁断の力＝**force-write（履歴改変）の戦闘版**。威力は**傷（自 repo の未解決 commit）の深さ**に比例。
- 代償：**呪い（供給網汚染パッケージ＝寄生する闇）に呑まれる**。乱用＝人格・repo の侵食。
- 物語的位置：**終盤の最大の見せ場かつ最大の倫理的危機**。append-only への侵犯＝世界最大の禁忌に触れる行為。主人公の異常 commit はこの領域に届きやすい（歩く exploit）。

## 7. 特訓（修行）の設計

特訓＝**より良い "現実のプログラマ" になる過程**。数値グラインドにしない。

| 特訓型 | 中身 |
|---|---|
| 依存グラフ読解 | 自作技の裏依存を可視化、無駄を削り同容量に倍積む（build 最適化） |
| 加速制御 | crash せず演算を焚く閾値の体得。焚きすぎ＝記憶欠落の恐怖 |
| コード錬磨 | 自作関数/クラスの精度・速度を上げる。修行の成果が repo に commit され技能化 |
| 積み上げ・睡眠 | 日中の特訓（未確定差分）を**睡眠で定着（commit）**。徹夜・crash で当日分が飛ぶ恐怖。FAB の最適化補助＝近道だが代償あり（§3.5） |
| ダンジョン徒弟 | 死者 master の repo で学び、自分の関数として再実装 |
| 観測戦 | コンステレーション展開・対衛星・デッドゾーンでの盲目戦 |
| 心意制御 | 呪いに呑まれず意志で規則を曲げる修行（最危険・最重要） |

**naro 原則**：序盤で「現代/前世知識を試して失敗する」描写を必ず入れる。成功体験だけにしない。

## 8. codex 因果レンズ（合格判定）

| 問い | 本システムの答え |
|---|---|
| 主人公の行動をどう制約するか | 容量・演算・探知・致死禁忌が常時の枷。力任せに勝てない |
| 私的な願いをどう社会的問題へ変換するか | 勝利＝現実 deploy＝他者の現実と記憶の書き換え＝**勝つこと自体が政治・倫理問題**になる |
| 初期 hook を越えてどう拡張するか | 個人戦 → 領域の deploy 権争い → クラウド/学派の覇権 → main への書込権＝世界支配（local→world） |

→ **戦闘が"インフラ摩擦"を生む**（codex setting-engine 仮説に合格）。

## 10. 設計原則チェック

- **インフレ回避**：強さ＝literacy（理解・応用・相性）。数値ラダー不採用。✅
- **git を表に出さない**：技名は自由・現象として殴る。git は深層の伏線。✅
- **致死の抑制**：通常戦は非致死（記憶＋現実喪失）、致死は心意級限定。✅
- **失敗の描写必須**：特訓・実戦で知識を試して失敗する場面を欠かさない。✅
- **naro 整合**：現象として提示、内部機構（git）は作者限定。✅
