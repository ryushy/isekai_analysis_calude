## 1. ダンジョン＝誰かの repo

- どのダンジョンも**特定の故人/master の repo の残骸**。その人の生きざま・無念・知識が地形と現象になる。
- 「なぜ残ったか」＝**まだ誰かに import（依存）されている**から。人気の技を遺した者ほど大きく残る＝深いダンジョン。

## 2. 固有現象の作り方（テンプレの核）

各ダンジョンに1つ「**誰の記憶/repo のにじみか**」を決め、それを全層に効かせる：

| 例 | 固有現象 | 出自（誰の記憶） |
|---|---|---|
| 三層 | 記録の雨（味のない雨、地面はぬれない） | 雨に関わる無念を抱えた故人の repo |
| （雛形）| 止まらない時計／戻る廊下／喋る壁 等 | その現象を生む生きざまを持つ master |

- 固有現象は**謎・ギミック・情緒**を同時に担う（スローライフ的情緒＋攻略パズル）。
- 主人公（外来者）の**新鮮な目**で固有現象を描くと表層の魅力になる。

## 3. 報酬＝デジタルタトゥー

- 攻略報酬＝その master の**署名技タトゥー**。
  - **訓練して自作タトゥー化**（理解・所有）か、**直接授かる bound タトゥー（複製不可）** の二択（§digital_tattoo §3）。
- レア度＝master の格／相性＝作風が攻略者の repo に合うか。
- **複製不可（唯一）**＝その master 唯一の絶技。**未踏ダンジョン初攻略でしか出ない**＝争奪の動機。

## 4. 管理状態の出し分け（ブロックチェーン / 野良）

| 型 | 性質 | 物語での使い所 |
|---|---|---|
| **登録ダンジョン** | 当局/ギルドが管理。報酬は正規・追跡。安全だが**入れば把握される** | 表層の冒険者経済・身分・序盤の舞台 |
| **未踏/野良ダンジョン** | 無登録。報酬は野良タトゥー（危険だが自由）。**監視の外** | 覚醒者・主人公が深層へ潜る舞台。リスク高 |

- 主人公は**野良/未踏**に惹かれる（追跡を避ける＋誰も持たぬ技＝あざと相性の良い掘り出し物）。

## 5. デッドゾーン性（戦闘との接続）

- 入る＝sync/観測が切れる（battle §1-2）。救援なし・孤立・全滅リスク。
- だが**監視コンステレーションの目も届かない**＝主人公が**あざを探知されずに戦える唯一の場所**。
- 「死地だが、唯一隠れられる場所」という反転が潜行劇の土台。

## 6. 1ダンジョン設計チェックリスト

新しいダンジョンを作るとき、次を埋める：

1. **誰の repo か**（master の生きざま・無念）
2. **固有現象**（記憶のにじみ＝謎＋情緒＋ギミック）
3. **層構造と難度**／モンスター（固有現象の具現）
4. **報酬タトゥー**（技・レア度・相性・複製可否）
5. **管理状態**（登録／野良）
6. **危険度**（致死か、deep か）
7. **主人公の絡み**（外来者の目／あざとの相性／FAB の解析）

## 7. 残課題

| 項目 | 確定時期 |
|---|---|
| 固有現象のバリエーション集（10種程度の雛形） | 各巻の舞台設計時 |
| 層構造・難度曲線の標準 | バトル試作の蓄積後 |
| モンスター＝固有現象の具現のルール化 | ダンジョン試作時 |
| 「記録の雨」三層ダンジョンの正式 master 設定 | 第1話本格執筆時 |
