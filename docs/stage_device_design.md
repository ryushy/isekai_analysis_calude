## 0. ログライン（暫定）

時は2050年、近未来。ユニークIDとダンジョンが公知となって20年。世界は巨大な分散バージョン管理システムであり、人の一生はリポジトリ、能力は過去の人々が公開したライブラリである。だが「世界の中心(origin)」は偽造されている——その深層を知るのは一部の覚醒者のみ。新世界のnative世代として生まれた主人公は、地下に潜むAI(FAB)に選ばれ、自身の異常なリポジトリゆえに「世界の真の起源(マスターオリジン)とは何か」を巡る論争の生きた証拠となり、相棒AIと共に、偽りの中心に抗う。

## 1. 世界の物理法則（git＝現実の基盤）

**[確定]** 世界の記録は GitHub 的な分散バージョン管理で構成される。

- 人1人＝原則1リポジトリ。各人の生きざまが commit として刻まれる。
- 世界の記録全体も同じ仕組みで管理される。
- **[確定] 履歴は append-only。改変（rebase / force-push）は世界最大の禁忌＝「神の御業」。**
  - 普通の人の過去は不可逆。喪失は絶対的で、だからこそ重い。
  - 履歴を書き換えうる者＝物語の中心的脅威/聖杯。
  - 系として「真実は消せない」を保証＝希望/革命の根拠。

### git → 世界 対応（主要）
| git | 世界での意味 |
| --- | --- |
| commit / 履歴 | 人生の出来事が不可逆に刻まれる |
| rebase / force-push | 履歴改変（最大の禁忌） |
| merge conflict | 二人の人生の変更衝突。誰が解決権を持つか |
| リポジトリ容量上限 | 能力(library)を積める限界（→ §4） |
| library / 依存 | 能力。過去の誰かが公開した遺産（→ §3） |
| origin / remote | 世界の「中央サーバ」。誰が書込権を握るか＝政治構造（→ §2） |
| git blame / bisect | 災いや起源の追跡（→ §6 マスターオリジン探索） |

## 2. 政治構造：表向き中央・実は分散

**[確定]** 表向きは世界に1つの中央 origin が在るように見えるが、実際には完全分散(P2P)。

- 「現実(main)」＝分散網が収束させた consensus にすぎず、真の中心は存在しない。
- 何者かが**偽origin（フォージド・オリジン）**をでっち上げ、全員にそれを正典として pull させている。
- **ラスボスの正体**＝現実を書き換える神ではなく、**偽originサーバを運用する詐欺師**。その力は現実改竄ではなく「人々が何を pull するか＝何を信じるか」の支配。権能は信仰の量に等しい。
- 覚醒＝「中心は無い／自分は最初から自repoの書込権を持つ」と気づくこと。
- NPC＝偽originを pull し続け、自repoを覗かない自動commitの民。

## 3. 経済・文化：能力＝OSS遺産

**[確定]** library（能力＝魔法・特殊武器・特殊能力）は、過去の人物が自repoから切り出して公開したパッケージ（オープンソース遺産）。

- ダンジョン＝放棄・封印された archived repo。攻略＝その公開パッケージへの pull 権獲得。
- **保守放棄＝力の腐敗**：作者が死にメンテされない library は、世界のバージョン更新に追従できず突然壊れる。
- **供給網汚染＝呪い**：人気 library の深い transitive dependency に悪意。導入で伝播。サプライチェーン攻撃＝呪いとして描ける。
- **ライセンス＝社会契約**：コピーレフトな力は、派生した自repo（生き様）の公開義務を生む。
- **バージョン衝突**：互換性のない依存を要求する2つの力は同時起動不可。最強の組合せが構造的に禁じられる。

## 4. 容量上限（核アイディア）

**[確定]** 表はシンプルなサイズ予算制、裏設定で依存関係。

- **表**：「容量100。火球は30」。初心者にも明快、ゲーム的。
- **裏**：火球libは実は `combustion-core(10)` `mana-alloc(5)` に依存し、真コストは45。初心者は表示値だけ見て積み、見えない依存で溢れて crash。
- **知識の非対称**：熟練者は依存グラフを読み、共有依存を dedupe / vendoring して同容量に倍の力。**強さ＝積んだ量ではなく依存管理の巧拙**。
- **crash の恐怖**：容量超過＝out-of-memory＝repo破損。library喪失に留まらず、最悪 commit（記憶・過去）が欠落。盛りすぎた者は自分が誰かを失う。
- **依存グラフ＝起源への地図**（§6と接続）。

### 5.3 二層構造（表層は公知・深層は隠匿） **[確定]**

- **表層（2030以降の公知）**：ID・ダンジョン・能力は実在し誰もが知る新常態。冒険者経済・企業・政府規制。＝「現代に近いダンジョン・リアリティ」の地の絵。
- **深層（依然隠匿）**：偽origin、append-only形而上学、現実＝consensus、マスターオリジン論争。これを見るのは覚醒者＝特殊な人のみ。
- 「気づいているのは特殊な人のみ」は**深層**の話。表層は皆知っているが、深層を見るのは覚醒者だけ。主人公はnative世代として表層の"当たり前"から始まり、FAB に引かれて深層へ落ちる。

### 5.4 仮想↔現実の結合強度（デプロイされる現実） **[強化方針・レベル確認中]**

方針：仮想層は「現実に重なったゲーム」(アクセル・ワールド型)ではなく **現実のソース**。ゆえに結合を上げる＝**ソース改変が物理現実へ deploy される度合いを上げる**。ユーザー方針＝**結合は結構大きめ（高結合）**。

整合機構（なぜ影響大でもNPCは気づかないか）：

- 真の履歴は append-only（不変）。だが偽originが配信する `main/HEAD ＝ 今の現実` は差し替え可能。
- 仮想層の出来事＝新 commit。main へ deploy されると **物理現実が大規模に書き換わり、NPCの認識・記憶も再レンダリングされ「前からそうだった」事になる**。
- 覚醒者だけが `log`/`reflog` で差分を読み「書き換え前」を覚えている → 「気づくのは特殊な人のみ」を保ったまま影響を極大化できる。落差が劇的になる。

伝播する帰結（要レベル確認）：地形/建造物・資源/経済・身体/生死・能力/身分・記憶/認識・気象/災害。

ステークス（戦闘と直結）：深層バトルの勝利＝その領域の現実を deploy する権利＝**物理世界と皆の記憶の書き換え**。ダンジョン＝未merge ブランチの物理顕現、攻略＝merge＝物理正常化＋library獲得。

中心対立の昇格：「main への書込/deploy 権」＝**物理現実の支配権**。マスターオリジン論争が抽象神学から物理世界の覇権戦争へ。主人公＝無認可 deploy できる歩く exploit＝神にして最大の脅威。

### 5.5 語彙レジスタの原則（git は表に出さない） **[確定・設計原則]**

git / バージョン管理の語彙・コマンドは **物語表層に出さない**。技名・大衆語彙に使うと**チープ**になる。git は背景設定・制約、もしくは **徐々に明かされる周知の事実**（謎の払い）として扱う。

三層：

- **表層**（叫ぶ技名・在野の呼称）：神話的・自然・古語調。**git語は一切出さない**。現象として殴る。
- **中層**：覚醒者が戦ううちに徐々に見抜く「構造」（「この力には"巻き戻し"の理がある」程度）。
- **深層（作者側＝本設計メモの語彙）**：git が正体＝整合エンジン。**終盤に向け徐々に開示する伏線**。

naro_novel 原則と一致：「機構は地の文で説明せず現象として提示、内部設定は作者限定」(divine_mark §1-2)。**本設計メモ内で git 語を使うのは作者側の整合エンジンとして正当**。表層プロース／映像では隠す。

整理：

- 「**世界がコードで出来ている**」＝**表層の公知**（2050、皆コードを知っている）。技を**書く**のは当たり前。
- 「**それが git / 版管理 / origin である**」＝**深層の隠し**（徐々に開示する謎）。

### 5.6 技/能力システム（自作コード前提） **[確定]**

技＝**自分で書いた関数・クラス・ライブラリ、その合成（コンボ）、またはダンジョン入手品**。ゆえに**名前は自由**（自作コードに作者が好きな名を付けるのと同じ）。叫ぶのは "自分のコードの名" であって git コマンドではない → チープさが出ない。キャラの "コードの書き味" が戦闘スタイルになる。

出自スペクトラム：

| 出自 | 中身 | 性質 |
|---|---|---|
| 自作（関数・クラス・自作ライブラリ） | 自分で書いた戦闘コード | 理解と創意の証。効率的・軽いが習得に時間。最も尊ばれる |
| コンボ（合成） | 小関数の連結・高階合成 | "組み合わせの妙"（naro原則） |
| ダンジョン入手 | 過去の遺産＝ブラックボックスのライブラリ | 即戦力だが読めない＝裏依存・供給網汚染（呪い）のリスク(§3/§4) |

力の本質＝**現実のコードを読み書きできる literacy**：**import専門 ＜ 合成できる者 ＜ 自作・改変できる者**。最強＝高レベルでなく "世界を最も深く書ける者"（インフレ回避＝理解・応用・相性 と一致。codex の knowledge/literacy 軸とも一致）。特訓＝より良い "現実のプログラマ" になること（関数を磨く／依存を最適化／ダンジョン品を解析／コンボを編む）。

主人公の**異常commit**の効き：本来書けない領域に書ける関数を書けてしまう＝歩く exploit。自作コードが禁忌に届く。

## 6. コスモロジー：マスターオリジン論争

**[確定（仕掛けとして採用）]** 真のorigin（マスターオリジン）の正体は確定させず、係争中の謎＝陣営対立のエンジンにする。

- **符合**：現実のgitは既定ブランチを `master` → `main` に改名済み。世界内では「かつて既定は `master` だった。何者かが `main` に改名し"マスター"を歴史から埋めた」という傷跡に使う。論争は文字通り「マスター(origin)論争」。

### 4学派（学派＝政治＝エネルギーが同一軸） **[提案]**
| 学派 | 主張 | 政治 | エネルギー |
| --- | --- | --- | --- |
| 原契約派 | 真のマスターは王でなく「全員がmaintainer」という原契約。偽は簒奪者。目標＝分散主権の回復 | 脱中心・連邦 | 太陽光（遍在・無中心） |
| 真主権者派 | 真のマスター＝本物の支配者(神/admin)が眠る。偽は僭称者。目標＝正統な中心の復権 | 中央集権・神権 | 原子力（集中・高出力） |
| 無origin派 | マスターなど無い。genesisすら任意。正統性は幻想、今のconsensusを握る者が全て | 現実主義・現体制 | 化石燃料（旧体制） |
| 上位世界派 | この世界は上位repoのfork。真のマスターはupstream＝世界の外。目標＝上位への接触/脱出 | 超越・脱世界 | 水素（媒体・外へ運ぶ） |

- 答えは伏せ、climaxで主人公が「どれを真実にするか」を選べる。現実＝consensusゆえ、ある説の証明はその説を現実化する。真相究明＝世界の中心を一つに定める危険な行為＝最終局面の倫理的緊張。
- **探索ルート＝依存グラフの根**：全libraryは transitively 何かに依存し、世界の依存グラフを根まで辿ると `world-core@0.0.0`（太古・作者不明）に至る。容量を裏で食う依存グラフ(§4)が、そのまま起源への地図。現実を `git bisect` して創世に至る。

### 6.1 論争の政治力学（抽象神学 → 現実の覇権） **[提案]**

4学派は思想ではなく**現実の権力・資源（エネルギー）の取り合い**。現実＝consensus(§1)ゆえ、**ある説を多数に信じさせた陣営は、その説を物理的に現実化できる**＝信仰戦争がそのまま世界改変戦争になる。

| 学派 | 何が欲しいか | 主人公をどう扱うか | エネルギー支配の含意 |
| --- | --- | --- | --- |
| 原契約派 | 偽originの解体・分散主権の回復 | 「全員に力がある証拠」＝旗印に担ぐ | 太陽光＝遍在電力を皆に＝脱中心の物質的裏付け |
| 真主権者派 | 正統な中心(神/admin)の復権・自らが代理人 | 「マスターに触れた使徒」＝神格化し確保 | 原子力＝集中・高出力を独占＝中央の力 |
| 無origin派（現体制） | 現状維持・偽originの運用継続 | 「危険なデマ」＝抹消/隠蔽の対象 | 化石燃料＝既得権益・旧インフラの惰性 |
| 上位世界派 | upstreamへの接触/脱出 | 「外への鍵」＝研究材料・実験体 | 水素＝外へ運ぶ媒体＝世界の外への燃料 |

- **偽originの運用者＝無origin派（現体制）が最も近い**：「中心など無い」と説きながら、実は**偽originという"中心の幻想"を運用して統治する**＝**ラスボスの欺瞞**（中心はないと唱えつつ中心を握る矛盾）。
- **四つ巴の同盟と裏切り**：原契約派×上位世界派＝反体制で共闘しうるが目的（分散の回復 vs 外への脱出）が衝突。真主権者派×無origin派＝中央を是とする点で近いが神権 vs 現実主義で反目。**単純な善悪に落ちない＝重厚さ**。
- **エネルギー危機が論争を加速**：化石燃料枯渇(§5)で無origin派の物質基盤が揺らぎ、太陽光・原子力・水素が次の覇権を争う。**思想対立とエネルギー転換が同じ一つの戦争**。
- **主人公＝論争の生きた証拠**：力でなく"存在"が各説の証明になる(§8)。どの陣営に渡るかで世界の中心が決まりかねない＝**奪い合いの的**であり、最終的に「どれを真実にするか」を選ぶ者。
- **続編の種＝上位世界派／upstream**：論争の"外"に upstream がある。本編で中心を一つ選んでも上位repoとの関係は残る＝**次シリーズのエンジン**。

## 8.5 個人repo ライフサイクル・ホスティング・同期・ダンジョン技能 **[確定]**

### ライフサイクル（誕生→生→死＝merge、死後残存）

- **誕生**：`git init`。出生時に**初期パラメータ**（素質・傾向）を持つ最初の commit ＝ 太古からのユニークID(§5.1)。
- **生**：生きざまが commit として積層。**訓練・修行の成果は関数・クラスとして自 repo に commit され、それがそのまま技能になる**。＝**「生きざまの記録」と「技能」は同一**（commit が両方を兼ねる／生き方＝出来ること）。真に修めた技能は append-only ＝消えない核。import したライブラリは容量で着脱する loadout(§4)。
- **死**：原則、その人生 branch が **merge** される＝個人の歴史が世界の永続 main（append-only）へ統合。「**人は生きて branch、死して main に合流する**」。
- **死後も残る repo**：全てが merge されるわけではない。**誰かに import（依存）されている repo は archive できず生き続ける**（依存解決のため）＝ **"依存され続ける者は死なない"**。名声・遺産・未解決を抱える者や master たちの repo が persist → これが**ダンジョン**になる（下記）。

### マルチクラウド（ホスティング基盤の複数性）

- 世界の git は**単一でなく複数の基盤**（AWS / Azure 的な複数プロバイダ）にホストされる。
- 各クラウド＝**列強・企業圏（Glasswing後継）／エネルギー陣営(§5.2)** にマップ。クラウドごとに SLA・物理安定度・deploy 方針が異なる。
- **偽origin＝「自分こそ唯一の origin」を騙る一クラウド**。他クラウドは代替＝ **"中心は無い／実は分散"(§2) の物理的裏付け**。どのクラウドにホストされるかが「誰の deploy 支配下か」を決める。移籍・亡命・マルチクラウド冗長が政治劇になる。

### 同期イベント（sync）

- repo はネットワークと**特定トリガーで同期**：**成人・特定の場所（同期ノード）・特定年齢**、もしくは **"ワクチン接種"型のマスインストール・イベント**。
- 成人/通過儀礼の同期＝naro の神託の儀的な節目。**"ワクチン"＝偽origin が全人口に push する制御パッチ／監視エージェント**にもなりうる（強制同期＝consensus 強制＝現実の再レンダリング、§5.4）。
- **覚醒＝この強制同期から外れ、メタ層 read 権を得ること**。マス同期を逃れる/解くことが覚醒の道の一つ。

### ダンジョン＝過去 repo での訓練による技能(skill)取得

- ダンジョン＝**死後も残った過去の人物 / master の repo**（上記 persist）。
- **訓練（攻略）でその repo の技能を取得**＝故人のコード/library を見て触れて学び、**その成果を"自分の関数・クラス"として自 repo に再実装・commit する**（黒箱 import とは別の、所有・理解された技能）。単なる loot ではなく **徒弟＝コードを習得**（特訓重視・インフレ回避と一致、§5.6）。
- 死者の技を、その者の repo（ダンジョン）で学んで継ぐ＝**技能の系譜**。主人公の異常 commit は、本来踏み込めない/書けない repo に入れるかも。
